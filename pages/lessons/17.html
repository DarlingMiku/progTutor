Тестирование в Python

Что такое тестирование?

Тестирование — это процесс проверки программы на корректность ее работы. Тестирование позволяет выявить ошибки в коде и убедиться в том, что программа выполняет свои задачи корректно.

Типы тестирования

Существует несколько типов тестирования, включая:

  • Юнит-тестирование: Тестирование отдельных функций или классов.
  • Интеграционное тестирование: Проверка взаимодействия между модулями.
  • Системное тестирование: Полная проверка всей системы.
  • Регрессионное тестирование: Проверка того, что изменения в коде не нарушили предыдущую функциональность.

Модуль unittest

Python предоставляет встроенный модуль unittest, который позволяет создавать и запускать тесты для ваших программ.

Вот как выглядит базовый пример юнит-теста:

import unittest
  
  # Функция для тестирования
  def add(a, b):
      return a + b
  
  # Класс с тестами
  class TestAddition(unittest.TestCase):
      def test_add_positive_numbers(self):
          self.assertEqual(add(2, 3), 5)
  
      def test_add_negative_numbers(self):
          self.assertEqual(add(-2, -3), -5)
  
      def test_add_zero(self):
          self.assertEqual(add(0, 0), 0)
  
  if __name__ == "__main__":
      unittest.main()

Запуск тестов

Тесты можно запустить через командную строку, используя команду python -m unittest. Эта команда выполнит все тесты в текущем файле или указанной директории.

python -m unittest test_file.py

Ассерты в тестах

В тестах часто используются различные ассерты для проверки ожидаемых значений. Например:

  • assertEqual(a, b): Проверяет, что a и b равны.
  • assertNotEqual(a, b): Проверяет, что a и b не равны.
  • assertTrue(x): Проверяет, что x истинно.
  • assertFalse(x): Проверяет, что x ложно.
